## Key Strength : Hyper-Modular Architecture
We separate **Strategic Intent** from **Mechanical Execution** through a **Parameter-Injection Pattern**. This enables real-time adaptation of the robotâ€™s playstyle without recompiling the core logic.

### 1. Strategy Layer (The Director)
Defines the high-level intent based on the match context.
* **Role**: Analyzes the environment and sets a **global mode** for the team.
  </br>
  
* **Example**: 
  > When **`Score < Opponent`** and **`Time Remaining < 2min`**,  
  > the system switches to **`ALL_OUT_ATTACK`**.

### 2. Tactics Layer (The Tuner)
Translates strategy into specific constraints via the **BehaviorTree Blackboard**.
* **Role**: Injects parameters (speed, aggression, thresholds) instead of hard-coding behaviors.
  </br>
  
* **Key Idea**: **Context-Aware Parameter Composition** (1:N Mapping) </br>
Same Strategy can produce different parameter compositions
based on `ScoreDiff`, `BallPosition`, etc.
  </br>
  
* **Example**: 
   * **`OFFENSIVE` Strategy**: *Parameter Composition Example*
     
     The **Tactic Selector** evaluates real-time game data to choose the best tactical mode:

    **[Tactical Rules Logic]**

    * **1. If `ScoreDiff <= -3` (Desperate)**
      * âž¤ **`TOTAL_ASSAULT`**
      * `Defense Line` â†’ **4.5m** (Stay at Opponent Goal)
      * `Kick Threshold` â†’ **0.5m** (Shoot on sight)

    * **2. If `Ball in Own Half` (Counter)**
      * âž¤ **`COUNTER_ATTACK`**
      * `Max Speed` â†’ **1.0** (Fast Break)
      * `Defense Line` â†’ **0.0m** (Push to Half-line)

    * **3. Otherwise (Standard Attack)**
      * âž¤ **`PRESSING`**
      * `Defense Line` â†’ **1.0m** (High Press)
      * `Kick Threshold` â†’ **0.3m** (Balanced)

### 3. Execution Layer (The Engine)
The Execution Layer receives a fully composed parameter map generated by the Tactics layer.
Nodes such as StrikerDecision do not encode tactical logic.
Instead, they interpret the incoming parameters and adjust
their internal behavior accordingly.

In other words, the Execution Layer answers *how to act*,
while all contextual decision-making lives upstream.

* **Role**: Executes the **"How"** based on the **"What"** provided by the Tactics layer.
  </br>
  
* **Example**: The Execution Layer consumes a **Composite Parameter Map** aggregated from multiple specialized sub-tactics.

  **[Parameter Aggregation Map]**
  > | Parameter Source | Contributing Parameters (`Key` : `Value`) |
  > | :--- | :--- |
  > | **`Tactic_Pressing`** | â€¢ `Chase Speed` : **1.0**<br>â€¢ `Aggression` : **High** |

  > | **`Tactic_Finishing`** | â€¢ `Kick Threshold` : **0.3m**<br>â€¢ `Aim Bias` : **Corners** |

  **[Execution Behavior]**
  The **`Chase`** node reads `Chase Speed`, ignoring the source logic.
  **`StrikerDecision`** independently adapts its shooting condition using `Kick Threshold`.
